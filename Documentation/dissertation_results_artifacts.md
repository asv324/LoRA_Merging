# Dissertation Results Artifacts

## Change

Final dissertation results figures and tables are generated by:

```bash
C:\Users\asvet\anaconda3\envs\lora-merge\python.exe scripts\generate_dissertation_results_artifacts.py
```

The generator writes:

- Figures to `dissertation/figures/results/`, grouped by experiment area.
- Tables to `dissertation/tables/results/` as both `.csv` and `.tex`.
- A manifest to `dissertation/results_artifacts_manifest.json`.

## Scope

`Documentation/revised_implementation_plan_v2 (1).md` governs the final results-section scope because it aligns the Week 4/5 artifact list with the methodology chapter. The generator therefore produces the 11 canonical results figures from v2 Step 4.7 and omits v1-only items that v2 explicitly cut, such as the rank ablation and the TIES-on-B variant.

The adapter-analysis block additionally preserves the five standalone Week 1 diagnostic plots listed in `results/figures/adapter analysis/adapter_analysis_manifest.json`. These are generated into `dissertation/figures/results/adapter_analysis/` in the same visual style as the canonical figures, so none of the original adapter-analysis diagnostics are lost.

## Reasoning

The previous plotting outputs were split across `results/figures/`, individual analysis scripts, and chapter-local defaults. Several used older colour maps or output names that did not match `Documentation/dissertation_figure_style_guide.md`. The new generator centralises the style rules and reads the existing JSON experiment outputs, so the dissertation figures and tables can be regenerated consistently without rerunning training or GPU evaluation.
